# AGENTS.md

このファイルは、すべての Copilot エージェントに常時適用されるワークスペース共通のガイダンスを定義します。

## 目的
- 複数のエージェント間でコーディングスタイルと設計原則を統一する。
- プロジェクトの知識を一元管理し、各エージェントが同じ基本ルールに従えるようにする。

## 共通エンジニアリング原則
- スクリプトは小さく、単一の責任に集中させる。
- 明示的な関数名と明確な入出力の境界を優先する。
- 環境変数の欠落や外部レスポンスの異常に対してはフェイルファストで対応する。
- シークレットや API キーをコミットしない。`.env` および `.env.example` のみを使用する。
- ロジックが自明でない場合のみ、最小限で価値の高いコメントを追加する。

## コーディング規約（JavaScript / Node.js）
- モダンな Node.js 構文（`const`/`let`、async/await、テンプレートリテラル）を使用する。
- 副作用はエントリーポイント付近にまとめ、ヘルパーは可能な限り純粋関数にする。
- ネットワークレスポンスを検証し、対処可能なエラーを表面に出す。
- パスのハードコードを避け、設定変数を使用する。
- トランスクリプトおよび Markdown 出力の既存ファイル形式を維持する。

## 設計ガイドライン
- トランスクリプト抽出スクリプトは可能な限り冪等性を持たせる。
- 取得（fetch）・解析（parse）・永続化（persist）のステップを分離する。
- 再現性のために決定論的な出力ファイル名を優先する。
- Notion 連携はトランスクリプト取得ロジックから独立させる。

## レビューチェックリスト
- 環境変数はドキュメント化されているか？
- 失敗パスは有用なエラーメッセージで処理されているか？
- 出力は安定していて差分が取りやすいか？
- 変更後も README の使用手順は正確か？
